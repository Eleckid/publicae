Users encounter a completely different navigation structure depending on the page they visit. There's no consistent "shell" or frame. This makes the platform feel like disconnected micro-sites stitched together.

#14
P0 Critical

Claimed NFTs Still Showing as Available

The map displays 21,732 NFT drop points. All markers are green (available) — no blue (claimed) markers are visible anywhere. The API /api/get-nft-drops.php returns isClaimed: false for every single drop. The backend never updates the claim flag after a successful claim.

// API analysis:
• /api/get-nft-drops.php → 21,732 drops returned
• Every drop: isClaimed: false
• Frontend code is correct: fillColor: isClaimed ? '#00aaff' : '#00ff00'
• Root cause: backend does not update isClaimed flag on claim
• Fix: update isClaimed = true in DB on claim, AND filter by wallet in API
#14b
P0 Critical

Wallet Connection Fails — No Claimed NFTs Visible

The “My NFTs” button shows “Please install MetaMask” instead of opening wallet connection. The backend endpoint /api/get-user-nfts.php returns a server error, making it impossible for users to see their claimed NFTs.

// API test:
• /api/get-user-nfts.php → {"success":false,"error":"Server error"}
• Root cause: PHP backend bug (likely SQL query failure for wallet → NFTs lookup)
• Fix: debug PHP endpoint, fix SQL query for wallet → NFTs association
#15
P1 Important

$STATE Balance Not Displayed

The gameplay page shows “Install MetaMask” instead of the $STATE token balance. The fetchSTATEBalance() function fails silently with no fallback message for the user.

// Expected behavior:
• If MetaMask installed → show $STATE balance
• If MetaMask not installed → currently shows error
• Fix: show fallback message “Connect wallet to view balance”

P1 Important

AR World on Separate Domain — No Integration

The AR World Explorer lives on pblcnft-claim.onrender.com, completely disconnected from the main platform (nwo.capital). No shared authentication, no shared navigation, no shared wallet state. Users must re-connect MetaMask separately.

// 06 — Recommendation

Framework Recommendation

Based on the audit findings, here is our architecture recommendation for the redesign.

✔ Recommended: React + Vite

  • ➤ Solves inter-page state inconsistency via global store
  • ➤ Component reusability eliminates layout fragmentation
  • ➤ SPA navigation feels native, no full page reloads
  • ➤ Wallet state (Web3) managed centrally via context
  • ➤ Better developer experience for 17+ pages

Current: Vanilla HTML/PHP

  • ✖ Each page is an independent .php file
  • ✖ No shared state between pages
  • ✖ Layout code duplicated/divergent
  • ✖ Web3 wallet connection lost on navigation
  • ✖ No component system = inconsistent UI

⚠ AR World Explorer — Priority Backend Fixes

  • ➤ P0: Fix /api/get-user-nfts.php backend endpoint (currently returns server error)
  • ➤ P0: Update isClaimed flag in get-nft-drops API after successful claims
  • ➤ P1: Show fallback message for $STATE wallet balance when MetaMask is not available
  • ➤ P2: Add id attribute to search input on gameplay page
