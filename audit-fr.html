Résumé Exécutif

La plateforme Publicae présente une identité visuelle forte avec son esthétique cyberpunk inspirée de Matrix. Cependant, l'audit révèle des incohérences critiques qui érodent la confiance utilisateur, une navigation fragmentée sur 4+ systèmes de layout différents, et des problèmes responsive significatifs sur mobile.

Problèmes Identifiés

16 problèmes classés par criticité. P0 = casse la confiance, P1 = dégrade l'UX, P2 = polish à faire.

⚠ P0 — Critique

#1
P0 Critique

Solde $STATE incohérent entre les pages

Le même utilisateur voit 3 soldes $STATE différents selon la page visitée. Cela détruit la confiance et donne l'impression que la plateforme est peu fiable ou exploitable.

// Feed / Profil / Dashboard :
409,707,719,481 $STATE

// Gains / Launchpad :
190,816,106,996 $STATE

// Exchange (on-chain) :
875,183,153,310.64 STATE
#2
P0 Critique

Valeurs négatives sur la page Airdrops

Plusieurs métriques affichent des nombres négatifs énormes, indiquant soit un bug d'overflow, soit une erreur de logique dans le calcul de distribution des airdrops.

// Pool historique :
-1,506,078,780,288

// Votre part potentielle :
-2,258,357,461,832

// Gains Airdrop :
-218,891,612,485
#3
P0 Critique

4+ systèmes de layout incompatibles

Les utilisateurs rencontrent une structure de navigation complètement différente selon la page visitée. Il n'y a pas de « shell » ou cadre cohérent. La plateforme ressemble à des micro-sites indépendants assemblés.

// Sidebar (Dashboard) ≠ Navbar (Profil)
// ≠ Header compact (Feed) ≠ Sans nav (Top Créateurs)
Résultat : les utilisateurs se perdent

⚠ P1 — Important

#4
P1 Important

Responsive mobile cassé sur le Feed

Sur iPhone (390px), le header est tronqué : montant $STATE coupé, boutons Connect Wallet et Profil invisibles, bouton « Post » masqué, onglet « Monetized » coupé.

#5
P1 Important

Navigation fragmentée (3 menus différents)

Le popup profil montre 7 liens, la navbar haute montre 6 items, la sidebar du dashboard montre 17 items. Aucune cohérence — certaines pages ne sont accessibles que depuis certains menus.

#6
P1 Important

Indicateur « En ligne » intermittent

Le badge vert « 1 online » apparaît sur Feed, Gains, Microprêts, Paramètres mais disparaît sur Staking. La présence inconsistante nuit à la preuve sociale.

#7
P1 Important

État du wallet incohérent

Le Dashboard montre le wallet connecté (0x7a3F...8d0A) tandis que les autres pages montrent « Connect Wallet ». L'état n'est pas partagé entre les navigations.

⚠ P2 — Amélioration

#8
P2 Amélioration

Pas d'états Loading / Erreur / Vide

Aucun skeleton loader, aucun message d'erreur, aucune illustration d'état vide. Seule exception : « No Holdings Yet » sur le Launchpad.

#9
P2 Amélioration

Cards de posts sans hiérarchie visuelle

Pas de séparation claire entre le contenu média, le texte et les métadonnées. Les infos utilisateur, timestamps et métriques se confondent.

#10
P2 Amélioration

Typographie monospace partout

Bien que cohérent avec le thème cyberpunk, l'utilisation exclusive de monospace réduit la lisibilité du contenu long et des commentaires.

#11
P2 Amélioration

Accessibilité absente

Pas d'indicateurs de focus visibles, pas de lien skip-nav, attributs ARIA minimaux, pas de support navigation clavier, pas de support reduced-motion.

#12
P2 Amélioration

Fond Matrix gêne la lisibilité

L'animation de pluie Matrix en arrière-plan, bien que thématique, réduit le contraste et la lisibilité sur les pages avec du contenu dense.

#14
P0 Critique

Les NFT claimés restent affichés comme disponibles

La carte affiche 21 732 points de drop NFT. Tous les marqueurs sont verts (disponibles) — aucun marqueur bleu (claimé) visible. L'API /api/get-nft-drops.php retourne isClaimed: false pour chaque drop. Le backend ne met jamais à jour le flag après un claim réussi.

// Analyse API :
• /api/get-nft-drops.php → 21 732 drops retournés
• Chaque drop : isClaimed: false
• Code frontend correct : fillColor: isClaimed ? '#00aaff' : '#00ff00'
• Cause racine : le backend ne met pas à jour le flag isClaimed au claim
• Correctif : mettre à jour isClaimed = true en BDD au claim, ET filtrer par wallet dans l'API
#14b
P0 Critique

Connexion wallet échoue — NFT claimés invisibles

Le bouton « My NFTs » affiche « Please install MetaMask » au lieu d'ouvrir la connexion wallet. L'endpoint backend /api/get-user-nfts.php retourne une erreur serveur, rendant impossible la consultation des NFT claimés.

// Test API :
• /api/get-user-nfts.php → {"success":false,"error":"Server error"}
• Cause racine : bug backend PHP (probablement requête SQL défaillante wallet → NFTs)
• Correctif : déboguer l'endpoint PHP, corriger la requête SQL d'association wallet → NFTs
#15
P1 Important

Solde $STATE non affiché

La page gameplay affiche « Install MetaMask » au lieu du solde de tokens $STATE. La fonction fetchSTATEBalance() échoue silencieusement sans message de repli pour l'utilisateur.

// Comportement attendu :
• Si MetaMask installé → afficher le solde $STATE
• Si MetaMask non installé → affiche actuellement une erreur
• Correctif : afficher un message de repli « Connectez votre wallet pour voir le solde »
#16
P1 Important

AR World sur un domaine séparé — Aucune intégration

L'AR World Explorer est hébergé sur pblcnft-claim.onrender.com, complètement déconnecté de la plateforme principale (nwo.capital). Aucune authentification partagée, aucune navigation commune, aucun état wallet partagé. Les utilisateurs doivent reconnecter MetaMask séparément.

#17
P2 Mineur

Champ de recherche sans attribut ID

Le champ de recherche sur la page gameplay n'a pas d'attribut id ni name, empêchant une gestion correcte du formulaire et l'étiquetage pour l'accessibilité.

Divergence des Valeurs $STATE

Le problème central de confiance : le même compte affiche des soldes différents selon la page.

state_audit.log — Analyse de la divergence des soldes
// ---- Source A : "Total gains WebApp" (engagement + jeu + tips + airdrops) ----
// Pages : Feed, Profil, Dashboard, Staking, Microprêts, Paramètres, Jeu
$ solde_a = 409,707,719,481 $STATE

// ---- Source B : "Gains engagement uniquement" (sans jeu, sans airdrops) ----
// Pages : Gains, Launchpad
$ solde_b = 190,816,106,996 $STATE

// ---- Source C : Solde wallet on-chain (données blockchain réelles) ----
// Pages : Exchange
$ solde_c = 875,183,153,310.64 STATE

// ---- Vérification ----
$ delta_ab = solde_a - solde_b
$ delta_ab = 218,891,612,485 // ≈ correspond à "Jeu: 402B + Airdrops: -218B"

// ---- Cause racine ----
DIAGNOSTIC : Chaque page interroge des colonnes/calculs différents.
SOLUTION : Un endpoint API unique comme source de vérité.
// 06 — Recommandation

Recommandation Framework

Basé sur les constats de l'audit, voici notre recommandation d'architecture pour la refonte.

✔ Recommandé : React + Vite

  • ➤ Résout l'incohérence d'état inter-pages via un store global
  • ➤ La réutilisabilité des composants élimine la fragmentation
  • ➤ Navigation SPA native, sans rechargements complets
  • ➤ État wallet (Web3) géré centralement via context
  • ➤ Meilleure expérience développeur pour 17+ pages

Actuel : HTML/PHP Vanilla

  • ✖ Chaque page est un fichier .php indépendant
  • ✖ Aucun état partagé entre les pages
  • ✖ Code de layout dupliqué et divergent
  • ✖ Connexion wallet Web3 perdue à la navigation
  • ✖ Aucun système de composants = UI incohérente

⚠ AR World Explorer — Correctifs Backend Prioritaires

  • ➤ P0 : Corriger l'endpoint /api/get-user-nfts.php (retourne actuellement une erreur serveur)
  • ➤ P0 : Mettre à jour le flag isClaimed dans l'API get-nft-drops après les claims réussis
  • ➤ P1 : Afficher un message de repli pour le solde $STATE quand MetaMask n'est pas disponible
  • ➤ P2 : Ajouter un attribut id au champ de recherche de la page gameplay
